# 数据Json文件规范

使用对象数组形式，每个事件对象包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `year` | Number | 是 | 年份。正数为公元后，负数为公元前（如 `-200`） |
| `month` | Number/String | 否 | 月份（1-12）。负数用于时间偏移但不显示 |
| `day` | Number/String | 否 | 日期（1-31）。负数用于时间偏移但不显示 |
| `title` | String | 是 | 事件标题，显示在详情面板。建议 ≤17 字 |
| `label` | String | 否 | 短标签，显示在时间轴卡片。建议 ≤8 字，为空时显示 `title` |
| `importance` | Number | 否 | 重要程度（0-7）。数值越大优先级越高 |
| `desc` | String | 否 | 事件简介，显示在大卡片。建议 25 字左右 |
| `detail` | String | 否 | 详细信息，显示在详情面板。支持换行 |
| `era` | String | 否 | 历史时期分类，如"危机纪元" |

**原则**：一切以最终显示效果为准，规范与效果冲突时优先效果。

## 字段细则

### 日期偏移

同一年内多个事件时间不明时，可用负数月/日进行时间轴偏移，偏移值不显示。

- 例：`2007` 年 `-6` 月的事件显示在时间轴 2007.06 位置，但卡片只显示 "2007"
- 偏移幅度根据事件顺序或重要程度调整，以各缩放层级显示清晰为目标

### 重要度（Importance）

| 等级 | 用途 |
|------|------|
| 7 | 里程碑事件，全局视图必显 |
| 5 | 重要事件，影响整体视图 |
| 3 | 放大后重点显示 |
| 1 | 相邻事件中较重要者 |
| 0 | 默认，普通事件 |

**注意**：半数左右事件应为 0，少数为 5-7，拉开差距。可 ±1 微调以优化显示。

### 文本字段

- **title** ≤17 字，确保详情面板不换行
- **label** ≤8 字，title 超8字时必须提供（英文数字两个算1字，空格不算）
- **desc** 25 字左右，若 title 已足够清晰可留空

## 示例

```json
[
  {
    "year": 1979,
    "month": 10,
    "day": 21,
    "title": "叶文洁收到三体回复",
    "label": "叶收到三体回复",
    "importance": 7,
    "desc": "叶文洁收到三体世界的警告信息，并回复",
    "detail": "凌晨0点，在向太阳发射信息不到9年后...",
    "era": "黄金时代"
  },
  {
    "year": 1967,
    "month": "",
    "day": "",
    "title": "叶泰哲死亡",
    "label": "",
    "importance": 0,
    "desc": "叶泰哲在批斗会上被打死",
    "detail": "叶泰哲在批斗会上被打死，留下了崩溃的妻子...",
    "era": "文革时代"
  }
]
```
